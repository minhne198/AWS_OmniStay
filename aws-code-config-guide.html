AWS OmniStay

Hướng dẫn cấu hình code để dùng AWS services

File này dùng sau khi đã tạo dịch vụ trên AWS Console. Mục tiêu là nối frontend/backend hiện có với Amazon RDS MySQL, ElastiCache Redis/Valkey, S3, CloudFront, ALB, EC2, IAM, SSM/Secrets Manager và CloudWatch. Không cấu hình Terraform hoặc CDK trong tài liệu này.

Mục lục 1. Luồng cấu hình 2. Biến môi trường backend 3. SSM và Secrets Manager 4. Cấu hình EC2 systemd 5. Cấu hình frontend 6. CloudFront và ALB 7. Health check và bằng chứng 8. CloudWatch logs 9. Checklist ảnh chụp 10. Lỗi thường gặp

Bước tổng quan

Luồng cấu hình từ AWS Console sang code

Không dùng Terraform
1. Lấy endpoint từ AWS Console

Ghi lại RDS endpoint, Redis primary endpoint, ALB DNS, CloudFront domain và bucket S3 frontend.

2. Lưu thông tin nhạy cảm

DB password và connection string nên đặt trong SSM Parameter Store hoặc Secrets Manager.

3. Gắn cấu hình vào EC2

Backend đọc cấu hình bằng biến môi trường ASP.NET Core, không hard-code trong source.

4. Cấu hình frontend

Khi chạy qua CloudFront cùng domain, đặt API_BASE là /api.

Code hiện tại đã hỗ trợ chạy local bằng InMemory database và chạy AWS bằng RDS MySQL/Redis thông qua cấu hình. Vì vậy có thể code trước, còn thao tác tạo AWS Console và chụp màn hình thực hiện sau.

Backend

Biến môi trường cần cấu hình cho API

EC2 + RDS + Redis

Backend dùng Database:Provider để chọn database provider. Local/test mặc định là InMemory. Khi deploy lên AWS, đổi sang MySql và cung cấp connection string RDS.

Biến môi trường Giá trị mẫu Vai trò
ASPNETCORE_ENVIRONMENT Production Chạy backend theo môi trường production.
ASPNETCORE_URLS http://0.0.0.0:8080 API lắng nghe port 8080 để ALB forward traffic.
Database__Provider MySql Bắt backend dùng Amazon RDS MySQL thay vì InMemory.
ConnectionStrings__HotelBookingDb server=<rds-endpoint>;port=3306;database=hotel_booking;user=<user>;password=<password>;SslMode=Required Connection string tới RDS MySQL.
ConnectionStrings__Redis <redis-primary-endpoint>:6379,abortConnect=false Kết nối ElastiCache Redis/Valkey để cache kết quả search.
Cache__SearchTtlSeconds 120 Thời gian giữ cache kết quả tìm phòng.
AWS__Region ap-southeast-1 Hiển thị region trong endpoint /health/aws.
AWS__S3FrontendBucket omnistay-frontend-<account-id> Ghi nhận bucket frontend để demo/health check.
AWS__CloudFrontDomain dxxxxx.cloudfront.net Ghi nhận CloudFront domain để demo/health check.
Cors__AllowedOrigins__0 https://dxxxxx.cloudfront.net Chỉ cần khi frontend và API khác origin. Nếu CloudFront route /api/* cùng domain thì không bắt buộc.
Ví dụ .env cho EC2, không commit file này 
ASPNETCORE_ENVIRONMENT=Production
ASPNETCORE_URLS=http://0.0.0.0:8080
Database__Provider=MySql
ConnectionStrings__HotelBookingDb=server=omnistay-mysql.xxxxxx.ap-southeast-1.rds.amazonaws.com;port=3306;database=hotel_booking;user=admin;password=CHANGE_ME_FROM_SECRET;SslMode=Required
ConnectionStrings__Redis=omnistay-cache.xxxxxx.cache.amazonaws.com:6379,abortConnect=false
Cache__SearchTtlSeconds=120
AWS__Region=ap-southeast-1
AWS__S3FrontendBucket=omnistay-frontend-123456789012
AWS__CloudFrontDomain=dxxxxx.cloudfront.net
Không đưa mật khẩu database, AWS access key, secret key hoặc connection string thật lên GitHub. Khi chụp màn hình, che password và secret value.

Secrets

Lưu cấu hình trong SSM Parameter Store hoặc Secrets Manager

IAM role đọc secret

Để thể hiện sử dụng AWS service rõ ràng, nên lưu các giá trị runtime trong AWS thay vì file local trên EC2. EC2 instance role đọc parameter/secret khi khởi động app.

Tên parameter/secret đề xuất Loại Nội dung
/omnistay/prod/db-connection SecureString hoặc Secrets Manager Connection string RDS MySQL đầy đủ.
/omnistay/prod/redis String Redis endpoint dạng host:6379,abortConnect=false.
/omnistay/prod/cloudfront-domain String CloudFront domain dùng để set health metadata hoặc CORS.
/omnistay/prod/frontend-bucket String Tên S3 bucket chứa frontend.
IAM role gắn vào EC2 cần quyền tối thiểu: ssm:GetParameter, secretsmanager:GetSecretValue cho đúng ARN, quyền ghi CloudWatch Logs, và quyền đọc S3 artifact nếu backend được tải từ S3.

EC2

Ví dụ cấu hình systemd cho backend

Port 8080

ALB target group nên forward HTTP port 8080 tới EC2 và health check endpoint /health. Backend ghi log ra stdout/stderr để CloudWatch agent hoặc journald thu thập.

/etc/systemd/system/omnistay-api.service 
[Unit]
Description=AWS OmniStay API
After=network.target

[Service]
WorkingDirectory=/opt/omnistay/api
ExecStart=/usr/bin/dotnet /opt/omnistay/api/HotelBooking.Api.dll
Restart=always
RestartSec=10
Environment=ASPNETCORE_ENVIRONMENT=Production
Environment=ASPNETCORE_URLS=http://0.0.0.0:8080
Environment=Database__Provider=MySql
Environment=ConnectionStrings__HotelBookingDb=server=RDS_ENDPOINT;port=3306;database=hotel_booking;user=admin;password=READ_FROM_SECRET;SslMode=Required
Environment=ConnectionStrings__Redis=REDIS_ENDPOINT:6379,abortConnect=false
Environment=Cache__SearchTtlSeconds=120
Environment=AWS__Region=ap-southeast-1
Environment=AWS__S3FrontendBucket=omnistay-frontend-ACCOUNT
Environment=AWS__CloudFrontDomain=dxxxxx.cloudfront.net

[Install]
WantedBy=multi-user.target
File service ở trên là mẫu dễ hiểu để demo. Khi làm thật, không nên đặt password trực tiếp trong systemd file. Hãy lấy giá trị từ SSM/Secrets trong user data hoặc script bootstrap, rồi export vào environment file có quyền đọc hạn chế.
Lệnh kiểm tra trên EC2 
sudo systemctl daemon-reload
sudo systemctl enable omnistay-api
sudo systemctl restart omnistay-api
sudo systemctl status omnistay-api
curl http://localhost:8080/health
curl http://localhost:8080/health/aws

Frontend

Cấu hình config.js cho local và AWS

S3 + CloudFront

Frontend không hard-code endpoint trong source chính. File frontend/public/config.js quyết định API base URL. Repo đã có thêm file mẫu frontend/public/config.aws.example.js.

Local development 
window.__APP_CONFIG__ = {
  API_BASE: 'http://localhost:5000/api'
};
AWS production qua CloudFront cùng domain 
window.__APP_CONFIG__ = {
  API_BASE: '/api'
};
AWS production nếu gọi ALB trực tiếp 
window.__APP_CONFIG__ = {
  API_BASE: 'http://omnistay-alb-123456.ap-southeast-1.elb.amazonaws.com/api'
};
Cách tốt nhất cho demo là dùng CloudFront phục vụ frontend từ S3 và thêm behavior /api/* forward về ALB. Khi đó trình duyệt chỉ thấy một domain CloudFront, ít gặp lỗi CORS hơn và thể hiện được CloudFront.

Routing

Nối CloudFront, ALB và API route

/api/*
  1. CloudFront default behavior trỏ tới S3 private bucket qua Origin Access Control.
  2. Thêm origin thứ hai là ALB DNS, protocol HTTP, port 80.
  3. Thêm behavior path pattern /api/*, origin là ALB.
  4. Allowed HTTP methods: GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE nếu muốn đầy đủ cho API.
  5. Cache policy cho /api/*: nên dùng policy không cache hoặc TTL thấp để booking/search cập nhật đúng.
  6. Origin request policy: forward query string để endpoint search nhận city/checkIn/checkOut/guests.
Luồng chạy đúng: Browser → CloudFront → S3 cho frontend, và Browser → CloudFront /api/* → ALB → EC2 backend → RDS/Redis.

Bằng chứng runtime

Health check cần chụp màn hình

API evidence

Backend có hai endpoint kiểm tra. Endpoint /health dùng cho ALB target group. Endpoint /health/aws dùng để chứng minh code đã đọc cấu hình AWS runtime.

Kiểm tra qua ALB hoặc CloudFront 
curl http://ALB_DNS/health
curl http://ALB_DNS/health/aws
curl https://CLOUDFRONT_DOMAIN/health/aws
Output mẫu của /health/aws 
{
  "environment": "Production",
  "region": "ap-southeast-1",
  "databaseProvider": "MySql",
  "redisConfigured": true,
  "redisConnected": true,
  "searchCacheTtlSeconds": 120,
  "s3FrontendBucket": "omnistay-frontend-123456789012",
  "cloudFrontDomain": "dxxxxx.cloudfront.net",
  "apiBasePath": "/api"
}
Nếu databaseProvider vẫn là InMemory trên AWS thì backend chưa dùng RDS MySQL. Nếu redisConfigured là false thì chưa truyền Redis endpoint vào app.

Observability

CloudWatch logs cần xem khi demo

Cache HIT/MISS

Khi Redis hoạt động, backend ghi log cache ở luồng search. Search cùng một điều kiện hai lần để thấy lần đầu MISS và lần sau HIT.

Log message cần tìm 
Hotel search cache MISS for Da Nang 2026-08-15 2026-08-18 2
Hotel search cache HIT for Da Nang 2026-08-15 2026-08-18 2
Dịch vụ Bằng chứng trong CloudWatch/AWS Console
ALB RequestCount, TargetResponseTime, HealthyHostCount.
EC2/ASG CPUUtilization, instance healthy trong target group.
RDS MySQL DatabaseConnections, CPUUtilization, FreeStorageSpace.
ElastiCache CacheHits, CacheMisses, CurrConnections, CPUUtilization.
CloudFront Requests, cache behavior /api/*, distribution domain hoạt động.

Mentor evidence

Checklist ảnh chụp cấu hình code với AWS

Chụp màn hình
Mục Ảnh cần chụp Ý nghĩa
Backend config EC2 systemd status hoặc environment file đã che secret. Chứng minh code đọc cấu hình AWS runtime.
RDS RDS endpoint và security group inbound từ SG-EC2. Chứng minh backend dùng database AWS.
Redis ElastiCache primary endpoint và CloudWatch CacheHits/CacheMisses. Chứng minh search cache dùng Redis/Valkey.
Health API Trình duyệt/curl gọi /health/aws thấy MySql và Redis connected. Chứng minh code đã nhận cấu hình service.
Frontend config config.js trên S3 có API_BASE: "/api". Chứng minh frontend gọi backend qua CloudFront.
CloudFront Behavior /api/* trỏ về ALB origin. Chứng minh CloudFront route cả frontend và API.
CloudWatch logs Log cache MISS/HIT hoặc request log API. Chứng minh app chạy thật trên AWS và có quan sát hệ thống.

Debug

Lỗi thường gặp khi nối code với AWS services

Runbook nhanh
Lỗi Nguyên nhân thường gặp Cách xử lý
/health/aws trả InMemory Chưa set Database__Provider=MySql. Kiểm tra systemd environment, restart service.
API không kết nối RDS Sai connection string, DB chưa tạo, SG-RDS chưa cho SG-EC2 port 3306. Kiểm tra RDS endpoint, security group, subnet, secret value.
Redis không connected Sai endpoint hoặc SG-Redis chưa cho SG-EC2 port 6379. Kiểm tra ElastiCache endpoint, security group và private subnet.
Frontend báo lỗi gọi API API_BASE sai hoặc CloudFront chưa có behavior /api/*. Kiểm tra config.js, CloudFront behavior, ALB target group healthy.
ALB target unhealthy Port sai, app chưa chạy, health check path sai. Đặt target group port 8080, path /health, kiểm tra systemctl status.
Không thấy CloudWatch log CloudWatch agent/IAM role chưa cấu hình hoặc log group sai. Kiểm tra IAM role, CloudWatch agent config, journald/stdout source.
Khi ba điểm sau cùng đúng, demo đã thể hiện được dịch vụ AWS chính: frontend tải qua CloudFront/S3, backend chạy sau ALB/EC2/ASG, dữ liệu đi vào RDS MySQL và search dùng Redis/Valkey cache.
